# CS230
CS 230 project designing a scalable, multi-platform architecture for Draw It or Lose It. Focuses on client-server design, REST APIs, and distributed systems. Includes cloud deployment strategies, security, and design patterns to support cross-platform performance and scalability.

Project Overview

The client for this project, The Gaming Room, wanted to expand their existing Android game Draw It or Lose It into a scalable, web-based, multi-platform application. The goal was to design a system that could support multiple users across different devices while maintaining performance, security, and consistency. This required a client-server architecture with support for distributed systems and future scalability.

One area I feel I did particularly well in was clearly explaining technical concepts in a way that connected back to the client’s needs. I focused on aligning each design decision with performance, scalability, and usability, which helped make the documentation more practical and relevant. Working through the design document also helped me better understand how planning architecture ahead of time makes development smoother and more organized, especially when considering things like memory usage, storage, and platform compatibility.

If I could revise one part of my work, I would improve the depth of my system diagrams and architectural explanations. While the concepts were correct, adding more visual detail and clearer connections between components would make the design easier to understand. I would also expand on trade-offs between different platform choices to strengthen the overall analysis.

When interpreting user needs, I focused on scalability, accessibility, and performance. Since the client wanted the game to run across multiple platforms, I designed the system to support distributed access and efficient resource management. Considering user needs is critical because it ensures the final product is not only functional but also usable and effective in a real-world environment.

My approach to software design involved breaking the system into smaller components, analyzing requirements, and selecting the best architecture to meet those needs. In the future, I would continue using strategies like modular design, design patterns, and cloud-based architecture planning. I would also spend more time evaluating trade-offs early in the process to make more informed design decisions.
